About Secure Token Generator

The Secure Token Generator creates cryptographically random tokens using crypto.getRandomValues(). Use these for API keys, session tokens, CSRF tokens, one-time passwords, webhook secrets, and any other security-sensitive random strings. Supports hex, Base64 URL-safe, alphanumeric, and numeric formats.

Frequently Asked Questions

Hex and Base64 URL-safe are the most common for API tokens. Hex is easy to read and debug. Base64 is more compact — 32 bytes in Base64 is 43 characters vs 64 in hex.

For most uses, 32 characters of hex (128 bits) provides more than enough entropy. Critical secrets like signing keys should be 64+ characters (256 bits).
